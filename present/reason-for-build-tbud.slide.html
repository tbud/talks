About me

I used scala, playframework, akka, slick etc. to write deploy tool bugatti from Oct 2013.

Because of scala's steep learning curve, we rebuild bugatti project for at least 5 times in the past one year. And only 3 employee could use scala language. It is hard to popularize scala in our company.

In this project, we build a proxy named spirit to implement our distributed network connection. The spirit with 7 files and less than 1k scala code need 6 seconds to compile, and one-jar size almost 20M.

I restart to find a light, faster, easy to leaning language. I found golang. But there isn't an application framework like I had used in scala. So I began using golang to write tbud in Jan 2015.

  • Pros in using scala

Pros in using golang

  • Easy to learn
  • Faster compile speed
  • Smaller package size
  • Easy to deploy
  • With wonderful tool in golang sdk
  • DRY
